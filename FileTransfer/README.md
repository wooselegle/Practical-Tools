# 📁 文件传输助手

一个纯局域网的文件/文字传输工具，无需互联网，无需公网IP。在一台设备上启动服务，同一局域网内的其他设备用浏览器访问即可传输文件和文字。

## 核心功能

- **📄 文件传输**：支持单个/多个文件、文件夹的传输
- **📂 文件夹浏览**：可浏览并选择整个电脑的文件夹（通过 `webkitdirectory`）
- **🎯 拖拽上传**：支持将文件或文件夹直接拖拽到指定区域
- **✅ 选择性下载**：接收端可勾选部分文件/文件夹，也可展开文件夹只下载部分文件
- **💬 文字传输**：发送和接收多行文字，支持任意长度文本
- **📜 文字历史（跨设备共享）**：所有收发文字自动保存在服务器端，关闭浏览器再打开历史仍在，不同设备共享同一份历史记录
- **📋 一键复制**：点击历史中的任意文字即可复制到剪贴板
- **🗑️ 单条删除**：文字历史支持逐条删除
- **🔒 管理员面板**：白名单管理、设备阻止/解封、设备重命名

## 文件列表

| 文件 | 用途 |
|------|------|
| `server.py` | Python HTTP 服务器（信号中转 + 文件服务 + API） |
| `index.html` | 前端页面（UI 结构） |
| `app.js` | 前端逻辑（连接、文件/文字传输、管理面板） |
| `style.css` | 界面样式 |
| `whitelist.json` | 白名单 IP 列表（自动生成） |
| `blocked_ips.json` | 被封禁的 IP 列表（自动生成） |
| `admin_password.txt` | 管理员密码（自动生成，可修改） |
| `text_history.json` | 文字历史持久化文件（自动生成） |

## 使用方式

### 1. 安装 Python

需要安装 Python 3.8 或更高版本：
https://www.python.org/downloads/

安装时**必须勾选** "Add Python to PATH"

### 2. 启动服务器

```bash
# Windows
py server.py

# 指定管理员密码（4位数字）
py server.py 0000
```

> **注意**：如果 `python` 命令无法正常工作，请改用 `py` 命令或安装官方Python。

### 3. 访问页面

启动后在终端或弹出的 GUI 窗口中会显示访问地址，例如：

```
http://192.168.1.100:8000
```

- **本机访问**：`http://127.0.0.1:8000`
- **同局域网其他设备**：使用浏览器访问上述地址即可（手机、平板、其他电脑均可）

### 4. 使用传输功能

不需要手动"连接"设备——所有访问同一页面的设备自动发现彼此：

- 在"文件传输"页面选择文件/文件夹或拖拽，点击发送给所有在线设备
- 接收端会自动显示文件，勾选后下载
- 在"文字传输"页面输入文字，点击发送，所有设备都能收到
- 点击历史文字可直接复制到剪贴板

### 5. 管理员功能

点击左侧导航栏的「⚙️ 后台管理」，输入管理员密码（启动时显示的4位数字），可以：
- 查看所有在线设备
- 启用/关闭 IP 白名单
- 添加/移除白名单 IP
- 阻止/解封特定设备连接

## 文字历史说明

- 所有文字历史存储在服务器的 `text_history.json` 文件中
- 任何设备发送或接收文字时，历史会同步到服务器
- 服务器重启后历史不会丢失
- 删除某条历史时，所有设备下次刷新后都会看到更新
- 清空历史会从服务器中删除全部记录

## 系统要求

- Windows / macOS / Linux
- Python 3.8+
- 现代浏览器（Edge、Chrome、Firefox、Safari 等）
- 设备需在同一局域网（同一 WiFi / 热点 / 网线连接）

## 注意事项

- 请确保防火墙允许 8000 端口通信（Windows 上首次启动会有提示）
- 传输大文件时请保持两台设备稳定连接
- 文件通过服务器中继传输，单次传输文件合计建议不超过 50MB（大文件建议分批次发送）
- 文字历史上限 200 条，超出后自动丢弃最早的记录
- 密码存储在 `admin_password.txt`，如需修改密码直接编辑该文件（必须是4位数字）

## 端口

默认使用 **8000 端口**。如需修改，编辑 `server.py` 中的 `HTTP_PORT = 8000`。
